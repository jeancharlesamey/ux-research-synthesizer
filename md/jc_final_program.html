The product: UX Research Synthesizer

A Next.js web app where you paste or upload raw interview notes. The MCP server processes them through Claude API — with RAG pulling from a knowledge base of UX research frameworks — and returns structured output: themes, verbatim quotes with attribution, severity ratings, and a downloadable report. Everything typed, everything testable, everything open-source.

Architecture

  • Frontend: Next.js App Router + Tailwind CSS — built entirely via Claude Code, no manual scaffolding
  • MCP server: TypeScript, 3 tools — synthesize_notes, extract_quotes, generate_report
  • RAG layer: Embed a small knowledge base of UX research frameworks (JTBD, affinity mapping, thematic analysis) into a vector store. Claude retrieves relevant framework context before synthesizing.
  • Function calling: Structured output schema — Claude returns typed JSON (not free text), so the frontend can render themes as cards, quotes as chips, severity as badges
  • Streaming: Stream the synthesis in real time so the UI feels alive, not frozen

Week-by-week sprint

  • Week 1 — Stack setup + MCP server: Bootstrap Next.js with Tailwind via Claude Code (npx create-next-app, then hand it to Claude Code). Build the TypeScript MCP server with 3 tools. Get the Figma → Claude Code → browser loop working once, even on a static page — this is Meaghan's core workflow, feel it in your hands.
  • Week 2 — RAG + function calling: Build the UX frameworks knowledge base (10–15 documents: JTBD, thematic analysis, affinity mapping, etc.). Embed via Claude's embedding API or a lightweight vector store (Supabase pgvector works, you already know it). Add function calling so Claude returns structured JSON — not prose — that the frontend renders as components.
  • Week 3 — Frontend + streaming: Build the actual UI in Tailwind via Claude Code. Input area, streaming synthesis panel, structured results (theme cards, quote chips, severity badges). Use Claude Code for all frontend polish — this is exactly how Meaghan works. Note every moment where Claude Code surprised you or frustrated you.
  • Week 4 — Ship + write: Open-source on GitHub. Write the README as if another designer will install it. Swift + Apple Intelligence — your learning path

You don't need to become a Swift engineer. You need to be able to ship a specific app with specific capabilities. This path is scoped to exactly that.

Prerequisites to confirm: Mac with Apple Silicon (M-series), Xcode 16+, macOS Sequoia 15.1+. Apple Intelligence on-device features require iPhone 15 Pro / iPhone 16, or any M-series Mac. Check this before month 3 starts.

Week-by-week learning path (Month 3)

Week 1 — SwiftUI fundamentals

Goal: build a working UI that accepts a screenshot input and displays text output. No AI yet.

  • SwiftUI View, @State, @Binding, VStack / HStack
  • Drag-and-drop with DropDelegate or .onDrop()
  • Async/await for loading states
  • Resource: Apple's "Introducing SwiftUI" (free, developer.apple.com)

Week 2 — Vision framework (on-device image analysis)

Goal: extract text and UI region bounding boxes from a screenshot entirely on-device.

  • VNRecognizeTextRequest — OCR on screenshots
  • VNDetectRectanglesRequest — detect UI element boundaries
  • VNImageRequestHandler — run requests on a CGImage
  • Resource: WWDC session "Explore Vision APIs" (free)
// Extract text from a banking UI screenshot — fully on-device
func extractText(from image: CGImage) async throws -> [String] {
  let request = VNRecognizeTextRequest()
  request.recognitionLevel = .accurate
  request.usesLanguageCorrection = true
  let handler = VNImageRequestHandler(cgImage: image)
  try handler.perform([request])
  return request.results?
    .compactMap { $0.topCandidates(1).first?.string } ?? []
}

Week 3 — Natural Language framework + Core ML

Goal: classify extracted text against your banking UX pattern library on-device.

  • NLTagger — tag parts of text (useful for extracting error messages, CTAs, labels)
  • NLEmbedding — semantic similarity between extracted text and known patterns
  • MLModel — load a Core ML model for custom classification
  • Pattern library: Start with a JSON file of 30–50 banking UX patterns. This is your domain expertise made machine-readable.
// Semantic similarity — is "Mot de passe incorrect" close to your error pattern?
let embedding = NLEmbedding.wordEmbedding(for: .french)
let distance = embedding?.distance(
  between: "mot de passe incorrect",
  and: "error message",
  distanceType: .cosine
) ?? 1.0
// distance < 0.3 → likely a match to your error state pattern

Week 4 — Apple Intelligence Writing Tools

Goal: use Apple Intelligence to generate the human-readable audit report from your structured findings.

  • WritingToolsCoordinator — invoke Apple Intelligence summarization on your findings
  • Understand the Private Cloud boundary: summarization may use Apple's servers for complex tasks, but no user data is retained
  • Handle availability gracefully — Apple Intelligence requires specific hardware; your app should degrade to template-based reports on unsupported devices
  • Resource: WWDC 2024 "Write better code with Apple Intelligence" + "Integrate Writing Tools" sessions
